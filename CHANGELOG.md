# Changelog

이 프로젝트는 PC(Windows)와 Android 두 플랫폼을 하나의 버전 번호로 관리합니다.
새 기능/수정 사항은 실제로 버전을 올리기 전까지 `[Unreleased]` 아래에 먼저 기록하고,
릴리즈 시점에 해당 섹션 제목을 버전 번호로 바꿉니다.

버전 번호는 `src/dashboard.cpp`의 "TEXT RPG vX.Y.Z" 문자열과
`android/app/build.gradle`의 `versionName`을 항상 같이 맞춥니다.
(안드로이드의 `versionCode`는 사람이 보는 버전과 무관한 Play Store 내부 일련번호라
업로드할 때마다 그냥 1씩 증가시키면 됩니다.)

## [1.1.0] - 2026-07-24

### 추가
- 장비 삭제 / 리롤 기능 — 쓸모없는 아이템(특히 낮은 옵션의 전설 등급)이 인벤토리
  칸만 차지하던 문제 대응. 삭제는 즉시 버림, 리롤은 골드를 내고 스탯 종류만
  다시 뽑음(등급/보너스%는 유지). 리롤 비용은 등급이 높을수록 커짐.
  (비공개 테스트 피드백 #4)
- 업그레이드 탭에 골드/경험치/드랍률 현재 배율을 업글·계승·장비 기여분으로
  분해해서 보여주는 표시 추가 (비공개 테스트 피드백 #2)
- 최초 실행 시 영웅 선택 화면에 "이 게임은 방치형 RPG" 온보딩 설명 추가
  (비공개 테스트 피드백 #1)
- 체력흡수(라이프스틸) 발생 시 회복량이 고정 텍스트로 묻혀서 안 보인다는 피드백
  대응 — 새로 회복이 발생한 틱마다 2초간 밝게 떴다가 옅어지는 팝업으로 표시
  (비공개 테스트 피드백 #3)

### 수정
- **[버그]** 프레스티지 계승 보너스(legacyBonusPct)가 스탯 계산에서 100배로
  과적용되던 문제. 37.5% 보너스가 "1.375배"가 아니라 "38.5배"로 들어가서
  프레스티지 한 번에 체력/방어력/공격력이 수십 배씩 폭증했음.
  (비공개 테스트 피드백 #9)
- PC 트레이 / 안드로이드 상단 상태바의 "작은 아이콘"이 기본(비프라이버시) 상태에서
  여전히 검 모양이었던 문제 — 런처/배지용으로 이미 바꿔둔 마름모(젬) 디자인과
  안 맞아서 통일감이 없었음. 기본 아이콘을 마름모로 교체(PC `app_fantasy.ico`,
  안드로이드 `ic_diamond_notif.png` 신규 추가), 프라이버시 모드일 때의 sync
  아이콘(원형)은 그대로 유지 — 마름모 ↔ 원형으로 토글되도록 통일.

### 밸런스
- 2차 특성이 1차보다 계수가 20~40% 낮게 설계되어 있어 "2차 투자가 손해"로
  느껴지던 문제 — 매칭되는 스탯 계수를 1차와 동일하게 맞춤 (체력 보너스는
  2차 고유 혜택으로 유지). (비공개 테스트 피드백 #5)
- 초반 생존력 보강 — 기본 최대체력 시작값을 200 → 350으로 상향 (스테이지당
  성장률은 그대로 유지, 초반 쿠션만 확보). (비공개 테스트 피드백 #6)
- 인벤토리가 가득 찬 상태에서 드랍 롤이 될 때마다 반복해서 뜨던 "가방 가득 참"
  푸시 알림 제거 — 대시보드(최근 이벤트)에는 계속 표시되지만 시스템 알림은
  더 이상 안 보냄. (비공개 테스트 피드백 #7)
- 레벨업 경험치 요구량 지수를 1.30 → 1.25로 완화 — 기존 곡선은 2차 특성을
  다 채우는 레벨 40 도달까지 약 20시간이 걸려 "떡벽"으로 느껴졌음. 1.25로
  낮추면 레벨 40이 약 4시간대로 줄어들어 하루 안에 첫 프레스티지가 현실적인
  목표가 되면서도, 후반(레벨 50+)은 여전히 오래 걸리는 지수 곡선을 유지함.
  (비공개 테스트 피드백 #8)
- 도적 "기습/치명적 기습" 특성이 이미 공속 배수가 적용된 전체 데미지를 통째로
  ×2 하던 방식 → "한 타 분량"만 추가로 더하는 방식으로 변경. 공속(타수 증가)과
  기습 확률이 곱연산으로 겹쳐서 도적만 딜이 눈덩이처럼 불어나던 구조적 불균형
  수정. (비공개 테스트 피드백 #12)

### UI
- 장비탭에서 리롤/삭제 버튼이 아이템 정보와 같은 줄에 있어 PC 창 폭(460)을
  넘어가 잘리던 문제 — 버튼을 다음 줄로 내려서 항상 폭 안에 들어오게 수정.
- **[버그]** 체력흡수(라이프스틸) 회복량이 정수 HP로 버림 처리되면서 데미지가
  작을 때(예: 14뎀 x 3% = 0.42) 매번 0으로 사라지던 문제 — 크리티컬/폭발
  데미지처럼 큰 값이 아니면 회복이 거의 안 되는 것처럼 보였음. 버려지는
  나머지를 `lifestealCarry`에 이월해서 누적되면 반영되도록 수정(내 체력
  표시 자체는 정수로 유지). 회복 팝업(초록 글씨)은 정수로 반영된 양이 아니라 이번 틱에
  실제 계산된 소숫점 값(`lastHealFrac`)을 그대로 보여주도록 변경 —
  0.42처럼 작은 값도 매 틱 계속 뜨는 걸 확인할 수 있음.
