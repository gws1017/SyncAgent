# 비공개 테스트 피드백 #1

수신일: 2026-07-15 (날짜는 정리 시점 기준, 실제 플레이 시점과 다를 수 있음)
출처: 비공개 테스트 참가자 1명

원문 그대로 옮기지 않고 카테고리별로 정리함. 각 항목에 종류 태그를 붙임:
- 🐛 **버그 의심** — 의도한 동작이 아닐 가능성이 높음, 코드 확인 필요
- ⚖️ **밸런스** — 수치/성장 곡선 조정 검토
- 🖥️ **UX/표시** — 정보 노출/설명 부족
- ❓ **확인 필요** — 재현/분석 먼저 해야 판단 가능

---

## 1. 온보딩 / 설명 부족 🖥️

- 게임 진행 방식에 대한 설명이 아예 없음. 앱을 켜면 갑자기 자동으로 진행되는데,
  "이게 방치형 게임이고 알아서 큰다"는 걸 알려주는 화면/툴팁이 없어서 당황스러움.
  → 최초 실행 시 짧은 온보딩 or 설명 툴팁 필요.

## 2. 스탯 증가율이 "현재 대비 얼마나"인지 안 보임 🖥️

- 골드/드랍률 증가 자체는 좋은데, 이게 기존 대비 몇 % 증가한 상태인지 표시가 없음.
- 요청 예시: "레벨 9인데 지금 골드 획득 +280%, 장비 합산 +310%" 처럼 **현재 배율을
  분해해서 보여주는 표시**가 있으면 좋겠다는 의견.

## 3. 합연산 밸런스 / 체력흡수(라이프스틸) ⚖️❓

- 전체적으로 스탯이 합연산이라 특정 스탯(라이프스틸 등) 하나의 가치가 지나치게
  커지는 느낌.
- 라이프스틸이 "받는 데미지 이상으로는 회복 안 되게" 캡이 걸려있는 건지 불명확.
  실제로 회복이 되는지도 체감이 잘 안 됨.
- 전투 방식이 "내가 먼저 때리고 맞는" 순서라서, 항상 내가 먼저 공격 → 상대 반격
  순으로 진행돼 사실상 풀피 상태에서 시작하는 일이 거의 없음 (풀피 검증이 잘 안 됨).

## 4. 장비 인벤토리 관리 — 삭제/리롤 기능 없음 ⚖️

- 쓸모없는 전설 등급이 나오면 장비 칸만 차지하고 최종 빌드를 원하는 대로 못 맞춤.
- **장비 삭제** 또는 **옵션 리롤** 기능이 있으면 좋겠다는 요청.
- 구체 사례: 스테이지 42까지 진행, 전설 등급 4개 획득. 특성 1(방어 특화) 하나만
  찍고 방어력 60인 상태에서 전설 아이템의 방어력 상승폭이 겨우 15%밖에 안 되는데도
  칸을 차지함 → 등급은 높은데 실제 가치가 낮은 아이템이 인벤토리를 낭비하게 됨.

## 5. 1차 특성보다 2차 특성 옵션이 약함 ⚖️❓

- (전사 기준으로 추정) 특성 1보다 특성 2 쪽 옵션이 전체적으로 더 약하게 느껴짐.
  의도된 밸런스인지 확인 필요.

## 6. 초반 생존력이 너무 약함("개복치") ⚖️

- 자동 진행 시뮬레이션 느낌인데, 초반 캐릭터 능력치가 너무 약해서 쉽게 죽는
  ("개복치") 느낌이라는 의견. 초반 구간 생존력 상향 검토.

## 7. 가방 가득 찬 상태에서의 드랍 알림이 과함 🖥️

- 인벤토리가 가득 찬 상태에서 파랑(희귀) 등급 이상 드랍이 "확률에 걸리기만 해도"
  알림이 뜨는 게 과하다는 의견 (현재는 실제로 드랍되어 놓친 경우에만 알림이 가도록
  되어 있는데, 유저 입장에서는 이게 너무 자주/불필요하게 뜨는 것처럼 느껴진 것으로
  보임 — 빈도/조건 재검토 필요).

## 8. 레벨업에 필요한 경험치가 너무 가파름 ⚖️

- "떡벽"이라는 표현. 특성 2를 다 찍고 프레스티지를 눌러보려 했는데 그 전까지
  경험치 요구량이 지나치게 가파르다는 의견.

## 9. 프레스티지 이후 스탯이 곱연산처럼 폭증함 🐛

**가장 우선순위 높은 버그 의심 항목.**

- 첫 프레스티지 이전: 전사 특성 30포인트 투자, 체력 800, 방어력 300 / 200
- 첫 프레스티지 이후: 체력 1만대, 방어력 14000 / 16000

이 정도의 증가폭은 계승 보너스(legacyBonusPct, 프레스티지 1회당 투자량 비례 가산)
설계 의도와 맞지 않아 보임. 덧셈이 아니라 곱셈으로 누적되고 있는 건 아닌지
`GameTick`의 보너스 적용 공식(특히 `PlayerBaseMaxHp`, `PlayerBaseDef`와
`legacyBonusPct` 곱해지는 부분) 확인 필요.

## 10. 스테이지 성장 공식 분석 (유저 추정) ❓

유저가 직접 관찰/추정한 내용, 검증 필요:

- 스테이지 클리어에 따른 스탯 증가가 곱연산으로 붙는 것 같음 (장비를 껴도
  공격력이 급격히 안 오르는 걸 보고 추정).
- 추정 공식: `총공격력 = 레벨비례 스탯 + 스테이지당 추가 스탯`
- 장비/업그레이드/특성이 정확히 "레벨비례 스탯" 쪽을 올려주는 게 맞는지 확인 필요
  (즉, 스테이지당 추가 스탯 쪽은 장비 투자로 못 건드리는 영역이라 장비를 껴도
  체감 상승이 적어 보이는 것일 수 있음).

## 11. 현재 메타: 공격속도 몰빵이 사실상 최종 빌드 ⚖️

- 위 9~10번 분석의 결과로, 현재 빌드 버전에서는 **공격속도 스탯을 도배하는 게
  사실상 종결 빌드**가 되어버림. 다른 스탯(공격력, 치명타 등)의 상대적 가치가
  낮아 보임.

## 12. 클래스 간 딜 배율 불균형 ⚖️

유저가 관찰한 클래스별 딜 계산 방식:

- **전사**: 공격력 × 공속 (가장 "정직한" 방식)
- **마법사**: 폭발(크리티컬) 안 터질 때 기본뎀 × 공속, 터질 때는 공속 소수점을
  올림/버림/랜덤으로 처리한 값에 폭발뎀을 곱해서 들어감
- **도적**: 기본뎀 × 공속 × 2, 거기에 "공속 스택당 추가타 확률"까지 겹쳐서 붙음

→ 클래스 간 딜 배율이 몇 배씩 벌어지는 구조라 밸런스 조정이 필요해 보인다는 의견.

## 13. 프레스티지 이후 스탯 상승 속도가 캐릭터 간에 비슷함 ❓

- 다른 클래스로 처음부터 다시 해본 게 아니라 확실친 않지만, 프레스티지로 스탯이
  오른 이후로는 세 스탯(체력/방어력/공격력으로 추정)이 미는 속도가 서로 비슷해
  보이는 게 신기하다는 관찰. (9번 버그와 연관 가능성 — 계승 보너스가 스탯 종류
  무관하게 동일 비율로 곱해지고 있다면 이런 현상이 나올 수 있음)

## 우선순위 제안 (정리자 의견, 추후 논의)

1. **9번(프레스티지 스탯 폭증)** — 버그일 가능성이 높아 최우선 확인 대상
2. **10~13번(성장 공식/클래스 밸런스)** — 9번 원인 파악과 함께 봐야 정확한 진단 가능
3. **4번(장비 삭제/리롤)** — 요청 빈도 높고 UX상 필요성 명확, 기능 추가 검토
4. **2번(배율 상세 표시), 1번(온보딩)** — UX 개선, 비교적 가벼운 작업
5. 나머지 밸런스 항목(3, 5, 6, 8, 11, 12)은 9~10번 원인 파악 후 함께 튜닝
